# Pantry 项目评审报告

## 一、多瓶使用功能 Bug 分析

### 当前实现的问题

**问题1：数据结构不匹配**
- 当前：只有一个 `activePercent` 字段（平均余量百分比）
- PRD要求：每瓶独立余量，如"60抽"和"80抽"两包同时使用

**问题2：显示逻辑错误**
```javascript
// 当前代码（第1121-1124行）
const obHTML=Array.from({length:item.openCount},(_,i)=>{
  const e=i>0&&item.activePercent<5;
  return `<div class="ob-chip${e?' ob-chip-e':''}"><div class="ob-dot${e?' e':''}"></div><span>${e?'近空':item.activePercent+'%'}</span></div>`;
}).join('');
```
- 所有chip都显示相同的 `activePercent`，无法区分每瓶的实际余量

**问题3：每日扣减逻辑错误**
```javascript
// 当前代码（第1503行）
const dPct=(item.dailyUse*days/item.packageSize)*100;
item.activePercent=+Math.max(0,item.activePercent-dPct).toFixed(1);
```
- 没有除以 `openCount`，应该每瓶都扣减 `dailyUse / openCount`

**问题4：拖拽交互缺失**
- 当前拖拽直接修改 `activePercent`，无法区分拖的是哪一瓶
- 需要支持点击chip后拖拽对应瓶子的余量

### 修复方案

**数据结构调整：**
```javascript
// 原结构
{
  activePercent: 68,  // 平均余量
  openCount: 2
}

// 新结构
{
  openItems: [60, 80],  // 每瓶的余量百分比数组
  openCount: 2  // 等于 openItems.length
}
```

**计算逻辑调整：**
- `getRemain(item)`：计算总余量 = sum(openItems.map(p => packageSize * p / 100))
- `getTotal(item)`：总余量 + 库存
- 每日扣减：`openItems.forEach(p => p -= dailyUse/openCount/packageSize*100)`
- 拖拽：传入 `openIndex` 参数，只更新对应瓶子的余量

**UI调整：**
- 详情页chip显示每瓶的实际余量：`openItems[i]`
- 点击chip后，主插画显示对应瓶子的SVG
- 拖拽主插画时，更新当前选中瓶子的余量

## 二、照片/订单识别实现方案

### 技术选型

**方案A：Tesseract.js（推荐）**
- 纯前端OCR，无需后端
- 支持中文识别
- 体积较大（~2MB），但可接受
- 识别准确度：中等（适合结构化文本如订单截图）

**方案B：百度/腾讯OCR API**
- 需要后端代理（避免API Key泄露）
- 识别准确度高
- MVP阶段可能过度设计

**方案C：混合方案**
- 简单场景用Tesseract.js
- 复杂订单截图可考虑API（二期）

### 实现步骤

1. **集成Tesseract.js**
   ```html
   <script src="https://cdn.jsdelivr.net/npm/tesseract.js@4/dist/tesseract.min.js"></script>
   ```

2. **照片识别流程**
   ```javascript
   async function recognizePhoto(imageFile) {
     const { data: { text } } = await Tesseract.recognize(imageFile, 'chi_sim+eng');
     // 解析text，提取字段
     return extractFields(text);
   }
   ```

3. **字段提取规则**
   - 品名：匹配DETECT_MAP关键词
   - 品牌：常见品牌库匹配
   - 规格：正则匹配 "数字+单位"（如"250ml"、"80抽"）
   - 数量：匹配"×数字"或"x数字"
   - 保质期：匹配日期格式（YYYY-MM-DD或中文日期）

4. **订单截图识别**
   - 识别多行商品信息
   - 每行提取一个商品
   - 批量创建多个Item

### 注意事项

- 识别失败时降级到手动输入
- 显示"识别中..."加载状态
- 识别结果可编辑（用户可修正）

## 三、代码拆分方案

### 文件结构

```
pantry/
├── index.html          # 主HTML，只包含样式和基础结构
├── js/
│   ├── shapes.js       # SVG形状定义（SHAPES对象）
│   ├── data.js         # 数据模型和存储（loadItems/save/DEFAULT）
│   ├── detect.js       # 自动分类（DETECT_MAP/detectItem）
│   ├── utils.js         # 工具函数（getRemain/getTotal/getDays等）
│   ├── render.js        # 渲染函数（renderDashboard/renderGallery/openDetail）
│   ├── drag.js          # 拖拽交互（setupDrag）
│   ├── sheets.js        # Sheet弹窗（openSheet/openTake/openPour等）
│   ├── delete.js         # 删除功能（openDeleteSheet/undoDelete）
│   ├── add.js           # 录入功能（doScan/confirmAdd）
│   ├── deduction.js     # 每日扣减（runDeduction）
│   └── recognition.js    # OCR识别（recognizePhoto/recognizeOrder）
└── css/
    └── styles.css       # 样式文件（可选，或保留在HTML中）
```

### 拆分原则

1. **按功能模块拆分**：每个文件负责一个功能域
2. **保持依赖清晰**：data.js → utils.js → render.js
3. **HTML中只保留样式**：作为"样式规范文档"
4. **使用ES6模块**：`<script type="module">` 导入

### 迁移步骤

1. 先创建文件结构
2. 逐个提取函数到对应文件
3. 测试每个模块功能
4. 最后整合到index.html

## 四、错误处理方案

### 需要处理的场景

**1. localStorage配额超限**
```javascript
function save() {
  try {
    localStorage.setItem('pantry_v4', JSON.stringify(items));
  } catch (e) {
    if (e.name === 'QuotaExceededError') {
      showError('存储空间不足，请删除部分物品或清理浏览器缓存');
    }
  }
}
```

**2. JSON解析失败**
```javascript
function loadItems() {
  try {
    const s = localStorage.getItem('pantry_v4');
    return s ? JSON.parse(s) : [...DEFAULT];
  } catch (e) {
    console.error('数据解析失败，使用默认数据', e);
    showError('数据已损坏，已恢复默认数据');
    return [...DEFAULT];
  }
}
```

**3. OCR识别失败**
```javascript
async function recognizePhoto(file) {
  try {
    const result = await Tesseract.recognize(file, 'chi_sim+eng');
    return extractFields(result.data.text);
  } catch (e) {
    showError('识别失败，请手动输入');
    return null;
  }
}
```

**4. 图片加载失败**
- SVG渲染失败时显示占位符
- 网络字体加载失败时使用系统字体

### 错误提示UI

保持简洁，使用现有的alert样式：
```javascript
function showError(msg) {
  const alert = document.getElementById('alert-banner');
  alert.className = 'alert red';
  alert.style.display = 'flex';
  document.getElementById('alert-text').textContent = msg;
  setTimeout(() => alert.style.display = 'none', 5000);
}
```

## 八、照片识别技术方案（已更新）

### OpenRouter API集成

**模型选择：**
- **Claude 3.5 Sonnet**（推荐）：多模态能力强，中文理解好，价格适中
- **GPT-4V**：备选方案，准确度高但价格较高

**实现流程：**
1. 用户拍照/选择图片
2. 显示"识别中..."（最多4秒）
3. 调用OpenRouter Vision API
4. 解析返回的JSON，提取字段
5. 自动填充表单，用户可编辑

**API调用示例：**
```javascript
async function recognizePhoto(imageFile) {
  const formData = new FormData();
  formData.append('image', imageFile);
  
  const response = await fetch('https://openrouter.ai/api/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${OPENROUTER_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      model: 'anthropic/claude-3.5-sonnet',
      messages: [{
        role: 'user',
        content: [
          { type: 'text', text: '识别这张图片中的商品信息，返回JSON格式：{"name":"品名","brand":"品牌","packageSize":250,"unit":"ml","qty":1,"expiryDate":"2026-12-31"}' },
          { type: 'image_url', image_url: { url: base64Image } }
        ]
      }]
    })
  });
  
  const result = await response.json();
  return parseRecognitionResult(result);
}
```

**订单截图识别：**
- 同样使用Vision API
- Prompt中明确要求识别多个商品
- 返回商品数组，批量创建Item

**平台特定优化：**
- 淘宝：订单号、商品名称格式较规范
- 京东：商品规格信息较完整
- 中免日上：进口商品，保质期信息重要
- 拼多多：价格信息突出，商品名称可能简化

**优化策略：**
- 针对不同平台的Prompt微调
- 识别结果后处理（品牌库匹配、规格标准化）
