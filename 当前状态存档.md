# 当前状态存档（技术快照）

> 用途：给后续 coding agent / 自己回顾时，快速对齐「此刻仓库里真实在跑什么」。  
> 更新说明：不含任何密钥；环境变量仅在 Vercel 配置。

---

## 1. 主入口与页面

- **推荐唯一主入口**：`pantry.html`（部署后访问：`…/pantry.html`）
- `pantry-v4_1.html`：历史副本，建议不再作为主开发入口；识别链路已与后端代理对齐，避免误开旧页导致行为不一致

---

## 2. 后端（Vercel Serverless）

目录：`api/`

| 路径 | 作用 |
|------|------|
| `api/families/ensure.js` | 匿名 `clientId` 确保/创建家庭并绑定 |
| `api/families/[familyId].js` | `GET` 拉取家庭数据；`PUT` 带版本号写入（409 冲突） |
| `api/families/[familyId]/join-token.js` | 生成加入家庭的 token（v2） |
| `api/families/join.js` | 新设备用 token 加入家庭（v2） |
| `api/openrouter.js` | 代理 OpenRouter，前端不持有平台 Key |

**已修复问题（重要）**：`api/openrouter.js` 曾存在重复 `export default handler` 导致线上 500；当前为单一实现，错误体统一为 `{ error: { message } }` 便于前端解析。

---

## 3. 数据库（Supabase）

- 脚本：`supabase/schema.sql`
- **表名带 `pantry_` 前缀**（与同库其他项目隔离）：
  - `pantry_families`
  - `pantry_clients`
  - `pantry_join_tokens`
  - `pantry_recognition_usage`
- 触发器：`trg_pantry_families_updated_at`（条件创建，避免误删提示）

---

## 4. 前端云同步（v1）

在 `pantry.html` 中：

- 本地生成并持久化 `clientId`
- 启动时 `POST /api/families/ensure` 绑定/创建家庭
- `save()` / `saveLocations()` 防抖同步到 `PUT /api/families/:familyId`
- 409 时拉取服务端数据并提示用户

---

## 5. AI 识别

- `config.js` 中 `OPENROUTER_CONFIG.apiUrl` 指向 **`/api/openrouter`**
- 用户**无需**在浏览器配置平台 Key（设置里旧入口可后续下线或标为高级调试）
