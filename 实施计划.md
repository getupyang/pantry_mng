# Pantry 项目实施计划

## 已确认需求

1. ✅ **每日扣减**：完全不做，移除相关代码
2. ✅ **照片识别**：OpenRouter API + Claude 3.5 Sonnet，4秒内返回
3. ✅ **订单截图**：支持淘宝/京东/中免日上/拼多多
4. ✅ **数据导出**：JSON格式下载

## 实施步骤

### Phase 1: 修复多瓶使用Bug（优先级最高）✅

**数据结构变更：**
```javascript
// 旧结构
{
  activePercent: 68,  // 平均余量
  openCount: 2
}

// 新结构
{
  openItems: [60, 80],  // 每瓶余量百分比数组
  openCount: 2  // 等于 openItems.length，保持兼容
  // 保留 activePercent 作为计算属性（向后兼容）
}
```

**需要修改的函数：**
1. `getRemain(item)` - 计算总余量
2. `getTotal(item)` - 计算总余量（含库存）
3. `getStatus(item)` - 判断是否低量（基于总余量）
4. `openDetail(id)` - 显示每瓶的chip
5. `setupDrag(el, id, isDetail)` - 支持选择哪一瓶拖拽
6. `openTake(itemId)` - 新瓶加入时 push(100)
7. `openRetire(itemId)` - 移除空瓶时 pop() 或删除指定索引
8. `refreshGCard(item)` - 更新卡片显示
9. `refreshDStats(item)` - 更新详情页统计

**UI变更：**
- 详情页chip显示每瓶实际余量：`openItems[i]%`
- 点击chip选中对应瓶子，主插画显示该瓶
- 拖拽主插画时，更新当前选中瓶的余量

### Phase 2: 集成OpenRouter API ✅

**API配置：**
- 模型：`anthropic/claude-3.5-sonnet`
- 超时：10秒（已从4秒调整，图片识别需要更长时间）
- API Key：存储在localStorage，默认使用config.js中的Key

**实现函数：**
1. ✅ `recognizePhoto(imageFile)` - 单张照片识别
2. ✅ `recognizeOrder(imageFile)` - 订单截图识别（批量）
3. ✅ `imageToBase64(file)` - 图片转base64（带预处理：压缩、格式转换）
4. ✅ `parseRecognitionResult(result)` - 解析API返回
5. ✅ `normalizeExpiryDate(dateStr)` - 规范化保质期日期格式
6. ✅ `preprocessImage(file)` - 图片预处理（HEIC转换、压缩）
7. ✅ `compressImage(img)` - 图片压缩（最大1280x1280，质量0.7，目标300KB）

**Prompt设计：**
- ✅ 照片识别：提取 name/brand/packageSize/unit/qty/expiryDate
- ✅ 订单截图：提取多个商品，返回数组
- ✅ 年份识别规则：明确说明28年→2028年，强调准确识别
- ✅ 日期规则：只有年月时使用月初第一日

**已实现功能：**
- ✅ 图片格式转换（HEIC→JPEG）
- ✅ 图片压缩优化（降低传输大小和API成本）
- ✅ 详细的日志输出（便于调试）
- ✅ API配置验证函数
- ✅ 网络错误处理
- ✅ 超时错误处理
- ✅ 保质期日期合理性检查

## 技术选型

**OpenRouter模型：**
- 主选：`anthropic/claude-3.5-sonnet`（多模态能力强，中文好）
- 备选：`openai/gpt-4-vision-preview`（准确度高但贵）

**API调用方式：**
- 直接前端调用（需要API Key）
- 或通过后端代理（更安全，但需要后端）

## 注意事项

1. **向后兼容**：旧数据可能只有activePercent，需要迁移逻辑
2. **性能**：openItems数组操作要高效
3. **用户体验**：4秒超时，超时后降级到手动输入
4. **安全性**：API Key存储要安全（localStorage，未来考虑加密）
