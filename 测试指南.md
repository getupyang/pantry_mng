## 测试方法

### 方法1：直接打开HTML文件（最简单）

1. **电脑浏览器测试**：
   ```bash
   # 在项目目录下，直接用浏览器打开
   open pantry-v4_1.html
   ```
   或者双击 `pantry-v4_1.html` 文件

2. **手机浏览器测试**：
   - 需要先启动本地服务器（见方法2）
   - 或者将文件上传到GitHub Pages等静态托管服务

### 方法2：使用本地服务器（推荐，支持手机测试）

#### 使用Python（Mac/Linux自带）

```bash
# 进入项目目录
cd /Users/getupyang/Documents/ai/coding/家庭收纳

# Python 3
python3 -m http.server 8000

# 或 Python 2
python -m SimpleHTTPServer 8000
```

然后在浏览器访问：
- **电脑**：`http://localhost:8000/pantry-v4_1.html`
- **手机**：`http://你的电脑IP:8000/pantry-v4_1.html`
  - 查看电脑IP：`ifconfig | grep "inet "`（Mac）
  - 确保手机和电脑在同一WiFi网络

#### 使用Node.js（如果已安装）

```bash
# 安装http-server（只需安装一次）
npm install -g http-server

# 启动服务器
http-server -p 8000
```

#### 使用VS Code Live Server插件

1. 安装 "Live Server" 插件
2. 右键点击 `pantry-v4_1.html`
3. 选择 "Open with Live Server"

### 方法3：部署到GitHub Pages（永久访问）

1. **创建GitHub仓库**：
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/你的用户名/pantry.git
   git push -u origin main
   ```

2. **启用GitHub Pages**：
   - 进入仓库 Settings → Pages
   - Source选择 main 分支
   - 访问地址：`https://你的用户名.github.io/pantry/pantry-v4_1.html`

## 测试步骤

### 1. 基础功能测试

- [ ] 打开应用，检查是否正常加载
- [ ] 查看仪表盘，检查数据是否正确显示
- [ ] 切换到物品库（画廊模式），检查物品卡片
- [ ] 点击物品进入详情页

### 2. 多瓶使用功能测试（重点）

- [ ] **测试场景1：查看多瓶物品**
  - 找到有 `openCount > 1` 的物品（如抽纸）
  - 进入详情页，查看使用区的chip是否显示每瓶的余量

- [ ] **测试场景2：选择瓶子**
  - 点击不同的chip，查看主插画是否切换显示对应瓶的余量
  - 查看余量数字是否更新

- [ ] **测试场景3：拖拽调整余量**
  - 选择一个瓶子
  - 拖拽主插画上下移动
  - 检查是否只更新当前选中瓶的余量
  - 检查其他瓶的余量是否不变

- [ ] **测试场景4：取出库存**
  - 找到有库存的物品
  - 点击"取出"按钮
  - 选择取出数量（如2瓶）
  - 确认后，检查使用区是否增加了对应数量的瓶子
  - 检查新瓶的余量是否为100%

- [ ] **测试场景5：移除空瓶**
  - 找到有多瓶且有空瓶的物品
  - 点击"移除空瓶"按钮
  - 确认后，检查空瓶是否被移除
  - 检查openCount是否正确减少

- [ ] **测试场景6：倒入补充装**
  - 找到有补充装的物品（如沐浴露）
  - 选择一个瓶子
  - 点击补充装"倒入"按钮
  - 确认后，检查是否只更新当前选中瓶的余量

## 常见问题

### Q: 手机无法访问？
A: 确保手机和电脑在同一WiFi网络，使用电脑的IP地址而不是localhost

### Q: 数据丢失？
A: 数据存储在浏览器的localStorage中，清除浏览器数据会丢失。建议定期导出数据备份。

### Q: API Key在哪里配置？
A: 目前API Key已配置在 `config.js` 中，照片识别功能还未实现，后续会添加配置界面。
