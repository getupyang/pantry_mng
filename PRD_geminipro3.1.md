# 产品需求文档 (PRD)：家用品余量管家

## 1. 产品定位
一款基于 AI 图像识别与物理世界隐喻的“极简”家庭物资管理工具。通过“拍立得”式的录入和“神仙水瓶子”式的可视化交互，解决家庭日常消耗品（洗护、纸巾、清洁剂等）管理繁琐、容易过期、忘记补货的痛点。

## 2. 核心痛点与解决方案
*   **痛点 1：录入成本极高。** 传统 App 需要手动输入名称、容量、保质期，用户难以坚持。
    *   **解决方案：AI 智能解析。** 支持拍摄实物图或上传电商订单截图。AI 自动提取名称、品牌、规格（单瓶容量）、单位、囤货数量，并结合知识库推算保质期和日均消耗量。
*   **痛点 2：库存状态不直观，只有 0 和 1。** 
    *   **解决方案：灰度余量与物理隐喻。** 引入“液位计/进度条”概念，结合时间流逝自动扣减。
*   **痛点 3：单瓶消耗与总囤货的认知冲突。** （如：一包抽纸 80 抽，但我买了 10 包）。
    *   **解决方案：双层库存模型。** 拆分为“当前使用余量（如剩 20 抽）”和“未开封囤货（如 9 包）”。

## 3. 核心功能模块

### 3.1 极简录入 (AI Input)
*   **输入源**：摄像头拍照 / 相册选择（支持电商订单截图）。
*   **AI 提取字段**：
    *   `name` (物品名称)
    *   `brand` (品牌)
    *   `category` (分类)
    *   `totalCapacity` (单件规格，如 500)
    *   `unit` (单位，如 ml, 抽)
    *   `backupQuantity` (购买总件数 - 1，即未开封囤货数)
    *   `dailyConsumptionRate` (预估日消耗量)
    *   `expiryDate` (保质期失效日)：
        *   **优先级 1**：用户手动录入/修改（最高优先级）。
        *   **优先级 2**：AI 识别图片中的明确日期（如包装喷码、订单文本）。
        *   **优先级 3**：Web Search 检索通用保质期进行推算。此时需在 UI 上显性提示用户“未识别到具体保质期，已自动推算”，保持极简体验的同时确保信息透明。

### 3.2 核心概念定义 (Terminology)
为避免歧义，明确以下两个时间概念的区分：
*   **保质期 (Expiry Date)**：物品本身的物理/化学失效日期（即过期变质的时间）。对应字段 `expiryDate`。
*   **用完日 (Depletion Date)**：基于当前余量和日均消耗量，预估物品被完全消耗殆尽的日期。由系统动态计算得出，不作为固定字段存储。

### 3.3 双层库存模型 (Two-Tier Inventory)
*   **当前余量 (Active Item)**：正在使用的那一瓶/包。基于日消耗量每天自动递减。用户可通过滑动“瓶身”手动校准。
*   **囤货库存 (Backup Stock)**：未开封的完整包装数量。
*   **状态流转**：当“当前余量”耗尽时，若“囤货库存” > 0，提示用户“开启新一瓶”（当前余量回满，囤货 -1）；若“囤货库存” = 0，则触发补货提醒。

### 3.3 智能告警 (Smart Alerts)
*   **余量告急**：当前余量 < 20% **且** 囤货库存 = 0 时，打上「补货」标签。
*   **临期提醒**：保质期剩余 < 30 天时，打上「临期」标签。

### 3.4 双视图展示 (Dual View Modes)
*   **画廊模式 (Gallery Mode)**：
    *   高颜值、低屏效。
    *   卡片式网格布局，采用拟物化“磨砂玻璃瓶”设计。
    *   支持**上下拖动**瓶内液体调整余量。
*   **效率模式 (Efficiency Mode)**：
    *   高密度、重效率。
    *   列表布局，采用横向极简刻度滑块。
    *   适合快速扫视全家物资状态。

## 4. 未来迭代规划 (Phase 2)
*   **电商打通**：基于淘宝/京东授权，自动同步订单，实现“零操作”入库。
*   **价格监控**：当标记为“需补货”的物品在电商平台降价时，推送购买建议。
*   **消耗模型自适应**：根据用户手动校准的频率和差值，机器学习自动修正 `dailyConsumptionRate`，越用越准。
